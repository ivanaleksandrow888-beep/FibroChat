# Авторизация, сессии и устройства

- Access token короткоживущий и хранится в памяти сервера/клиента.
- Refresh token позволяет восстановить access token.
- Серверная сессия привязана к доверенному устройству.
- После F5 приватный bundle восстанавливается из `sessionStorage`.
- После полного закрытия браузера bundle открывается шестизначным PIN из локального зашифрованного vault либо паролем аккаунта.
- PIN локальный: он не заменяет пароль и не передаётся серверу.
- Отзыв устройства завершает его сессии.
