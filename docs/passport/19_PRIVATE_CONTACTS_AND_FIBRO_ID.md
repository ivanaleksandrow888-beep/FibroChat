# Private Contacts and Fibro ID — v0.5.1

## Принцип
FibroChat не публикует общий каталог участников. Контакт появляется только на основании доверительной связи: приглашение, административная иерархия, точное добавление по Fibro ID или существующая переписка.

## Видимость
- Пользователь: пригласивший, добавленные контакты, собеседники.
- Администратор: суперадминистратор, приглашённые им пользователи, добавленные контакты, собеседники.
- Суперадминистратор: администраторы, приглашённые им пользователи, добавленные контакты, собеседники.

## Fibro ID
Формат: `FIBRO-XXXX-XXXX-XXXX`. ID создаётся сервером, уникален в пределах сети, не зависит от никнейма и не предназначен для частичного поиска.

## API
- `GET /api/contacts` — только разрешённые контакты.
- `POST /api/contacts/add` — точное добавление по полному Fibro ID.
- `GET/POST /api/messages` — разрешены только для контактов.

## База данных
Миграция `003_private_contacts_fibro_id.sql` добавляет коллекцию `contacts` и уникальный индекс Fibro ID. Существующим пользователям ID назначаются при запуске. Поле `invitedBy` восстанавливается из использованных инвайтов, когда это возможно.
