# FibroChat v0.7.0-alpha1

Первая альфа-версия ветки 0.7 добавляет инфраструктуру вложений.

## Реализовано

- API `POST /api/attachments?recipientId=...` для загрузки непрозрачного бинарного контейнера.
- API `GET /api/attachments/:id` с проверкой доступа владельца и получателя.
- Клиентское AES-256-GCM шифрование файла до отправки на сервер.
- Ключ и IV файла передаются только внутри существующего E2E-зашифрованного сообщения.
- Выбор файла, отображение карточки, скачивание и локальная расшифровка.
- Лимит вложения 10 МБ по умолчанию; на сервере настраивается через `MAX_ATTACHMENT_BYTES`.
- Метаданные вложений в JSON fallback и PostgreSQL.
- Миграция `006_attachments.sql`.
- Протокол FibroChat 1.2 и capability `attachments.opaque.v1`.

## Ограничения alpha1

- Один файл на сообщение.
- Нет превью изображений и индикатора процентной загрузки.
- Нет удаления осиротевших файлов после неудачной отправки сообщения.
- Резервная копия содержит метаданные, но бинарные файлы каталога `data/attachments` нужно копировать отдельно.
