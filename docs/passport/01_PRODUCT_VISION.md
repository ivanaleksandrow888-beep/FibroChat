# Видение продукта

FibroChat должен быть независимым защищённым мессенджером, который владелец сети может развернуть на своей инфраструктуре. Приоритеты: контроль данных, понятные роли, защищённые устройства, прозрачный деплой и возможность дальнейшей федерации нескольких узлов.

## Неприкосновенные принципы
- Пароли не хранятся открытым текстом.
- Приватные ключи не отправляются серверу.
- Администратор не получает права суперадминистратора автоматически.
- Обновление приложения не должно удалять PostgreSQL.
- Каждый релиз включает код, миграции, changelog, паспорт и инструкцию отката.
