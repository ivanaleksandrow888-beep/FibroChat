# Fibro Protocol 1.1

## Назначение

Fibro Protocol задаёт совместимый формат общения браузерного клиента, головного узла и будущих клиентов для Android/Windows/Linux.

## Согласование версии

Клиент отправляет:

- `X-Fibro-Protocol: 1.1`
- `X-Fibro-Client-Version: 0.3.0`

Сервер отвечает:

- `X-Fibro-Protocol: 1.1`
- `X-Fibro-Server-Version: 0.3.0`
- `X-Trace-Id: <UUID>`

Несовместимая основная версия получает HTTP `426` и код `PROTO_INCOMPATIBLE`.

## JSON-метаданные

Каждый обычный JSON-ответ содержит `_protocol`:

```json
{
  "version": "1.1",
  "serverVersion": "0.3.0",
  "traceId": "UUID",
  "time": "ISO-8601"
}
```

## Событийный конверт

```json
{
  "protocol": "1.1",
  "type": "message:new",
  "traceId": "UUID",
  "timestamp": "ISO-8601",
  "payload": {}
}
```

## Возможности узла

Список доступен через `GET /api/protocol`. Клиент не должен предполагать наличие функции, если capability отсутствует.

## Правило совместимости

- изменения внутри версии `1.x` должны быть обратно совместимыми;
- удаление или переименование обязательного поля требует версии `2.0`;
- неизвестные дополнительные поля должны игнорироваться;
- содержимое Secure Envelope не расшифровывается головным узлом.
