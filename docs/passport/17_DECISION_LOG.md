# Журнал решений

1. Пользовательские данные перенесены из JSON в PostgreSQL.
2. Dockerfile является production-способом сборки; Nixpacks не используется.
3. Единственный суперадминистратор управляет подтверждением и ролями; администраторы могут выдавать инвайты.
4. PIN локальный и не заменяет пароль аккаунта.
5. QR в будущем подтверждает одноразовую сессию нового устройства и не должен содержать приватный ключ.
6. Интерфейс 0.5.0 разделяется без немедленного разрушительного переписывания API и всего `app.js`.
7. Каждый релиз должен включать актуальный паспорт.
